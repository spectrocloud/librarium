<Tabs groupId="registry-type">

<TabItem value="internal-zot-registry" label="Internal Zot Registry">

1. Log in to the Local UI of the leader host of the {props.version} management cluster.

2. From the left main menu, click **Content**.

3. Enter the filename of the uploaded pack in the **Filter by name** search bar. The pack should appear in the table
   below. You can repeat this step for each pack you uploaded.

</TabItem>

<TabItem value="external-registry" label="External Registry">

1. List the packs uploaded to your external registry using the Palette CLI. Replace `<registry-dns-or-ip>` with the DNS/IP address of your registry and `<registry-port>` with the port number of your registry. If you have changed the base content path from the default, replace `spectro-content` with the correct content path.

   If you are using custom TLS certificates or choosing to skip TLS, use the appropriate flags as shown in the following examples.

   <Tabs groupId="tls-palette-cli">

   <TabItem label="Custom TLS Certificate" value="custom-tls-certificate">

   ```shell
   palette content list \
   --repo <registry-dns-or-ip>:<registry-port>/spectro-content \
   --ca-cert <path-to-ca-cert> \
   --tls-cert <path-to-tls-cert> \
   --tls-key <path-to-tls-key>
   ```

   </TabItem>

   <TabItem label="Skip TLS" value="skip-tls">

   ```shell
   palette content list \
   --repo <registry-dns-or-ip>:<registry-port>/spectro-content \
   --insecure
   ```

   </TabItem>

   </Tabs>

2. Check that the pack bundles you uploaded are listed in the output as repositories.

</TabItem>

</Tabs>
