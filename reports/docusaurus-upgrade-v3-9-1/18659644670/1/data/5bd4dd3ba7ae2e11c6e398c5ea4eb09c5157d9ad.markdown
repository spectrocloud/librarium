                - heading "Airgap Resources" [level=1] [ref=e269]
                - paragraph [ref=e270]: You can install Palette in an airgapped environment. An airgap environment lacks direct access to the internet and is intended for environments with strict security requirements.
                - paragraph [ref=e271]: The installation process for an airgap environment is different due to the lack of internet access. Before the primary Palette installation steps, you must download the following artifacts.
                - list [ref=e272]:
                  - listitem [ref=e273]:
                    - paragraph [ref=e274]: Palette platform manifests and required platform packages.
                  - listitem [ref=e275]:
                    - paragraph [ref=e276]: Container images for core platform components and third-party dependencies.
                  - listitem [ref=e277]:
                    - paragraph [ref=e278]: Palette packs.
                - paragraph [ref=e279]: The other significant change is that Palette's default public OCI registry is not used. Instead, a private OCI registry is utilized for storing images and packs.

                - paragraph [ref=e283]:
                  - img "An architecture diagram outlining the five different install phases" [ref=e284]
                - list [ref=e285]:
                  - listitem [ref=e286]:
                    - paragraph [ref=e287]: Download the airgap setup binary from the URL provided by the support team. The airgap setup binary is a self-extracting archive that contains the Palette platform manifests, images, and required packs. The airgap setup binary is a one-time use binary for uploading Palette images and packs to your OCI registry. You will not use the airgap setup binary again after the initial installation. This step must be completed in an environment with internet access.
                  - listitem [ref=e288]:
                    - paragraph [ref=e289]: Move the airgap setup binary to the airgap environment. The airgap setup binary is used to extract the manifest content and upload the required images and packs to your private OCI registry. Start the airgap setup binary in a Linux Virtual Machine (VM).
                  - listitem [ref=e290]:

                  - listitem [ref=e293]:
                    - paragraph [ref=e294]: Install Palette using the Palette CLI or the Kubernetes Helm chart.
