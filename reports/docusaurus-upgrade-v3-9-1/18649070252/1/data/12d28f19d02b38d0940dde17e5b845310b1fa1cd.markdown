                - heading "Enable Overlay Network" [level=1] [ref=e348]
                - paragraph [ref=e349]: Edge clusters are often deployed in locations where network environments are not managed by teams that maintain the Edge deployments. However, a Kubernetes cluster, specifically several control plane components, requires stable IP addresses. In the case of an extended network outage, it is possible that your cluster components would lose their original IP addresses when the cluster expects them to remain stable, causing the cluster to experience degraded performance or become non-operational.
                - paragraph [ref=e350]: Palette allows you to create a virtual overlay network on top of the physical network, where the virtual IP addresses of all cluster components are managed by Palette. Inside the cluster, the different components use the virtual IP addresses to communicate with each other instead of the underlying IP addresses that could change due to external factors. If the cluster experiences an outage with the overlay network enabled, components inside the cluster retain their virtual IP addresses in the overlay network, even if their IP addresses in the underlying physical network change, protecting the cluster from an outage.
