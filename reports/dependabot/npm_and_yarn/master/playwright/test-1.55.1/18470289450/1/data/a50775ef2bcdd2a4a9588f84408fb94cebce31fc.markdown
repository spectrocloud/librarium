                - heading "Airgap Installation" [level=1] [ref=e281]
                - paragraph [ref=e282]: You can install Palette in an airgap VMware vSphere environment. An airgap environment lacks direct access to the internet and is intended for environments with strict security requirements.
                - paragraph [ref=e283]: The installation process for an airgap environment is different due to the lack of internet access. Before the primary Palette installation steps, you must download the following artifacts.
                - list [ref=e284]:
                  - listitem [ref=e285]:
                    - paragraph [ref=e286]: Palette platform manifests and required platform packages.
                  - listitem [ref=e287]:
                    - paragraph [ref=e288]: Container images for core platform components and third-party dependencies.
                  - listitem [ref=e289]:
                    - paragraph [ref=e290]: Palette packs.
                - paragraph [ref=e291]: The other significant change is that Palette's default public OCI registry is not used. Instead, a private OCI registry is utilized for storing images and packs.
                - heading "OverviewDirect link to Overview" [level=2] [ref=e292]:

                - paragraph [ref=e294]: Before you can install Palette in an airgap environment, you must complete all the required pre-installation steps. The following diagram outlines the major pre-installation steps for an airgap installation.
                - paragraph [ref=e295]:
                  - img "An architecture diagram outlining the five different installation phases" [ref=e296]
                - list [ref=e297]:
                  - listitem [ref=e298]:
                    - paragraph [ref=e299]: Download the airgap setup binary from the URL provided by the support team. The airgap setup binary is a self-extracting archive that contains the Palette platform manifests, images, and required packs. The airgap setup binary is a one-time use binary for uploading Palette images and packs to your OCI registry. You will not use the airgap setup binary again after the initial installation. This step must be completed in an environment with internet access.
                  - listitem [ref=e300]:
                    - paragraph [ref=e301]: Move the airgap setup binary to the airgap environment. The airgap setup binary is used to extract the manifest content and upload the required images and packs to your private OCI registry. Start the airgap setup binary in a Linux Virtual Machine (VM).
                  - listitem [ref=e302]:
                    - paragraph [ref=e303]: The airgap script will push the required images and packs to your private OCI registry.
                  - listitem [ref=e304]:
                    - paragraph [ref=e305]: Install Palette using the Palette CLI or the Kubernetes Helm chart.
