# Certificates

VMO Manager integrates with the platform's certificate infrastructure for TLS and CA trust. Certificate management is available under **Settings > Certificates**.

## Platform CA

The platform uses a central CA (Certificate Authority) for signing TLS certificates across services. When deployed with cert-manager, the platform CA is typically:

- Auto-generated by cert-manager
- Stored in a ConfigMap or Secret in the cluster
- Used to sign the Traefik ingress TLS certificate and other service certs

VMO Manager reads the platform CA from the configured path (default: `/etc/ssl/custom/ca.crt`) and uses it to verify connections to the Kubernetes API, CDI upload proxy, and other in-cluster services.

> **Note:** All TLS connections use the platform CA trust store. `InsecureSkipVerify` is not used in production code paths.

## Adding Trusted CAs

You can add additional trusted CA certificates for services that use certs not signed by the platform CA (e.g., corporate proxies, external registries).

### Upload PEM

1. Go to Settings > Certificates.
2. Click **Add Trusted CA** or **Upload**.
3. Paste or upload a PEM-encoded certificate (`.crt` or `.pem`).
4. Provide a descriptive name for the certificate.
5. Save.

The certificate is stored in the VmoTrustedCert CRD and added to the application's CA pool at startup. Newly added CAs require a pod restart to take effect.

### Viewing Certificate Details

For each trusted CA, the Certificates page displays:

| Field | Description |
|-------|-------------|
| Name | Admin-assigned label |
| Subject | Certificate subject (CN, O, etc.) |
| Issuer | Certificate issuer |
| Not After | Expiration date |
| Fingerprint | SHA-256 fingerprint for verification |

### Deleting a Trusted CA

1. Go to Settings > Certificates.
2. Locate the CA in the list.
3. Click the delete action.
4. Confirm.

The CA is removed from the VmoTrustedCert store. A pod restart may be required for the change to take effect.

## Platform CA from ConfigMap

When the platform CA is provided via a cert-manager ConfigMap (e.g., `ca.crt` in a well-known ConfigMap), VMO Manager loads it from the path specified by the `CA_FILE` environment variable. This value is read-only in the Configuration page and is set by the Helm chart or deployment manifest.

## TLS Certificate Management

VMO Manager can manage the TLS certificate used by the Traefik ingress for the platform. This is the certificate presented to users when they access the platform URL.

### Viewing TLS Certificate

The Certificates page shows the current TLS certificate details:

- Subject, Issuer, SANs (Subject Alternative Names)
- Expiration date
- Whether the certificate is managed by cert-manager (read-only) or manually uploaded

### Uploading a Custom TLS Certificate

If the platform is not using cert-manager for TLS, you can upload a custom certificate and private key:

1. Go to Settings > Certificates.
2. Select the TLS certificate section.
3. Upload the certificate (PEM) and private key (PEM).
4. Save.

> **Warning:** Uploading a custom TLS certificate may overwrite the existing Traefik TLS secret. Ensure you have a backup and that the certificate is valid for your platform hostname or IP.

### cert-manager Integration

When cert-manager manages the platform TLS certificate, the Certificates page displays it as read-only. Renewals are handled by cert-manager. VMO Manager does not modify cert-manager-managed certificates; it only displays their status and details.
